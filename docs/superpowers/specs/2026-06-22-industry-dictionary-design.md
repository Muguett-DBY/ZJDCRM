# 行业字典管理设计

## 目标

让唯一 `admin` 在「系统管理 → 字典配置」维护招商线索的行业下拉选项；新增和编辑线索读取启用的行业项，无需再次部署。

## 数据与兼容性

新增默认字典 `industry`，预置现有的医疗器械、医药健康、AI/人工智能、集成电路、智能制造和其他六项，编码保持现有 `industry_code` 值。既有企业记录继续保留原行业编码；编辑历史记录时，即使该编码已经停用，表单仍显示该值，避免数据无法保存。

## 管理界面与权限

字典配置页为每个字典项提供新增、编辑、停用/启用、删除按钮。删除采用软删除；行业表单只加载状态为 `active` 的项目。所有写操作限制为唯一系统管理员，要求 CSRF，并写入审计日志。

## 接口

- `GET /api/dictionaries/:code/items`：认证用户读取一个启用字典的启用项，按排序返回。
- `PUT /api/admin/dictionaries/:dictionaryId/items/:itemId`：更新名称、展示值、排序或状态。
- `DELETE /api/admin/dictionaries/:dictionaryId/items/:itemId`：软删除未被移除的字典项。

## 验证

集成测试覆盖行业字典读取、管理员编辑和删除；浏览器测试覆盖管理员新增行业后，新建线索表单显示该选项。
